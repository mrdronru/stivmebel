# Спецификация баннера для Telegram-поста
## Стив Интерьеры

---

## Размеры

- **Формат:** 1080 × 1350 px (4:5, вертикальный пост)
- **Инструмент рендеринга:** `wkhtmltoimage` (HTML → PNG → JPG)
- **Финальный формат:** JPG, quality=92

---

## Фоновое фото

- Любое вертикальное фото работы (шкаф, кухня, гардеробная)
- Если фото горизонтальное — обрезать центральную вертикальную полосу через Pillow
- Фото должно быть заранее подготовлено в размере **1080 × 1350 px** и сохранено рядом с HTML-файлом
- Фото подключается как `background-image: url('имя_файла.jpg')` на `.post` div
- `background-size: cover`, `background-position: center top`

---

## Затемнение (darklayer)

- Абсолютный div поверх фото
- `top: 780px` — начало затемнения (примерно на уровне нижней трети фото)
- `height: 570px` — до самого низа (780 + 570 = 1350)
- Градиент сверху вниз:
  ```css
  background: linear-gradient(
    to bottom,
    rgba(15,13,11,0) 0%,
    rgba(15,13,11,0.72) 18%,
    rgba(15,13,11,0.90) 32%,
    rgba(15,13,11,0.90) 100%
  );
  ```
- **Важно:** `top` подбирается под конкретное фото — затемнение должно начинаться ниже заголовка и полностью накрывать подзаголовок и всё что ниже

## Что менять при новом баннере

1. **Фоновое фото** — подготовить 1080×1350, сохранить рядом с HTML
2. **Заголовок** — текст в `.headline`
3. **Подзаголовок** — текст в `.sub`
4. **Кнопка** — текст в `.cta-button`
5. **Позицию `top` у `.darklayer`** — подобрать под конкретное фото (затемнение должно начинаться ниже заголовка, но накрывать подзаголовок)

---

## Команды для рендеринга

```bash
# Рендер PNG
wkhtmltoimage --width 1080 --height 1350 --enable-local-file-access banner.html render.png

# Сжатие в JPG через Python
python3 -c "
from PIL import Image
img = Image.open('render.png').convert('RGB')
img.save('telegram_post.jpg', quality=92, optimize=True)
"
```

## Технические ограничения wkhtmltoimage

- `object-position` на `<img>` тегах **не работает** — использовать `background-image` на div
- При повторном рендеринге с тем же именем файла движок может кэшировать — менять имя выходного файла при итерациях
- Шрифт Arial доступен в системе, Georgia тоже. Сторонние шрифты (Jost, Cormorant) не подключать — нет сети
